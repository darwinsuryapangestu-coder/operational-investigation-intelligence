# Operational Investigation Intelligence  
By: Darwin Surya Pangestu

---

## Problem

Industrial systems (PI, DCS, SAP, CMMS) produce a lot of data.  
But engineers still need days or weeks to explain why an incident happens.

The problem is not data.

The problem is:

> turning operational data into trusted reasoning.

---

## Context (Confidential)

- PI System
- DCS
- SAP / CMMS
- Reliability systems

All data is confidential (NDA).

---

## Workflow

1. Get historical data (PI)
2. Check anomalies
3. Check alarms / events (DCS)
4. Check maintenance history (SAP)
5. Build hypotheses
6. Validate evidence
7. Make conclusion

---

## Key Insight

This process is slow not because data is missing.

But because:

> context is fragmented and must be manually connected.

---

## Value

- Faster investigation
- Faster root cause understanding
- Faster decision support

Target:
> days/weeks → hours

---

## Positioning

Not dashboard work.  
Not reporting work.

This is:

> operational investigation and reasoning support.

---

## Objective

Reduce time from:

> question → trusted answer
